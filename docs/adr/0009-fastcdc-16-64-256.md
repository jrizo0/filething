# FastCDC 16/64/256 KiB para sync de código

filething sincroniza código, no backups, así que partimos los archivos con FastCDC en normalized chunking nivel 2 y parámetros `min`=16 KiB / `avg`=64 KiB / `max`=256 KiB (ratio 1:4:16), no el ~256K/1M/4M típico de un backup tipo restic. La gear table se siembra con el chunk secret aleatorio por-Space (no derivado de la Space key, ver ADR 0010), de modo que rotar la Space key no re-chunkea el mundo y los cortes son impredecibles entre Spaces.

## Considered Options

- **avg ~1 MiB (estilo backup, rechazado):** con ese avg, todo archivo de fuente < 256 KiB sería 1 solo Block; editar 1 byte re-subiría el archivo entero y mataría el delta intra-archivo. Con 16/64/256 hay delta real (editar una línea re-chunkea solo 1-2 Blocks) y el conteo de objetos sigue acotado por `min`.

## Consequences

- El dedup NO cruza Spaces: el mismo archivo en dos Spaces produce cortes y `cid` distintos. Aceptado y coherente con scope de dedup = Account/Space, no global.
- Mitiga el fingerprinting de tamaños de objeto (canal lateral), al ser los cortes secretos por-Space.
